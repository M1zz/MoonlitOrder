# 달빛 결사 (Moonlit Order)

아이폰끼리 **MultipeerConnectivity**(같은 Wi-Fi / 블루투스 근거리 통신)로 직접 연결해서 즐기는
5~10인용 소셜 추리 보드게임 앱입니다. 인터넷 서버 없이, 방에 모인 사람들끼리 바로 플레이합니다.

테마·역할 명칭·문구는 전부 오리지널 창작물입니다. (게임 *메커니즘* 자체는 저작권 보호 대상이
아니지만, 특정 상용 게임의 이름·일러스트·고유 캐릭터명은 쓰지 않도록 만들었습니다.)

---

## 게임 규칙 요약

- **진영**: 달빛 결사(선) vs 그림자(악). 그림자 단원들은 서로를 알고, **예언자**(선)는 그림자가 누군지 압니다.
- **진행**: 총 5번의 원정(미션). 매 라운드 리더가 원정대를 지명 → **전원 찬반 투표**(공개, 동수는 부결) →
  가결되면 원정대원이 몰래 **성공/실패 카드** 제출.
- **승리 조건**
  - 달빛 결사: 원정 3회 성공. 단, 성공하더라도 마지막에 **암살자**가 예언자를 정확히 지목하면 역전패!
  - 그림자: 원정 3회 실패, 또는 원정대 구성 **5회 연속 부결**, 또는 예언자 암살 성공.
- **인원별 구성**: 5~6명 → 그림자 2명 / 7~9명 → 3명 / 10명 → 4명.
  7명 이상일 때 4번째 원정은 실패 카드가 2장 나와야 실패합니다.

## 연결이 끊겨도 괜찮아요

- 게임 상태는 **호스트 기기**가 전부 들고 있고, 매번 전체 상태를 다시 보내줍니다.
- 각 기기는 고유 플레이어 ID를 저장하고 있어서, 앱이 튕기거나 연결이 끊겨도
  **앱을 다시 열면 같은 자리로 자동 복귀**합니다 (역할·투표 상태 그대로).
- 누군가 끊기면 화면 상단에 "재접속 대기 중" 배너가 뜨고, 돌아오면 자동으로 이어집니다.
- 누군가 영영 돌아오지 않으면 호스트가 좌측 상단 ✕ → **"게임 중단하고 대기실로"**로
  게임을 리셋하고 남은 인원으로 새로 시작할 수 있습니다.
- 단, 호스트 기기는 게임의 서버 역할이므로 게임 중 앱을 끄면 안 됩니다.
  (플레이 중에는 화면 자동 잠금이 꺼지도록 해 두었습니다.)

## 실행 방법

1. **Xcode 16 이상**에서 `MoonlitOrder.xcodeproj`를 엽니다.
2. 프로젝트 설정 → *Signing & Capabilities*에서 본인의 **Team**을 선택하고,
   *Bundle Identifier*(`com.example.MoonlitOrder`)를 본인 것으로 바꿉니다.
3. 아이폰을 연결하고 실행(⌘R)합니다. 시뮬레이터는 Multipeer가 제한적이므로 **실기기**를 권장합니다.
4. 처음 실행 시 **"로컬 네트워크 접근 허용"** 팝업이 뜨면 반드시 **허용**을 누르세요.
   (실수로 거부했다면: 설정 → 개인정보 보호 → 로컬 네트워크에서 다시 켜기)

### 함께 플레이하기

1. 한 명이 **방 만들기(호스트)** — 이 기기가 게임 서버가 됩니다.
2. 나머지는 **게임 참가** → 목록에서 호스트의 방을 탭.
3. 5명 이상 모이면 호스트가 **게임 시작**.
4. 각자 몰래 역할 확인 → 리더가 원정대 지명 → 전원 투표 → 원정 진행!

> Wi-Fi와 블루투스가 켜져 있어야 서로를 찾을 수 있습니다. 같은 Wi-Fi에 있으면 가장 안정적입니다.

### 참고: 인원수와 연결 한도

Apple의 MultipeerConnectivity는 세션당 8대(호스트 포함)가 공식 권장 한도입니다.
이 앱은 호스트가 세션을 여러 개 운용하는 방식으로 9~10명도 수용하지만,
기기·네트워크 환경에 따라 8명 초과 시 연결이 불안정할 수 있습니다.

## 프로젝트 구조

```
MoonlitOrder/
├── MoonlitOrderApp.swift        앱 진입점 + 화면 라우팅
├── Models/
│   ├── GameModels.swift         진영·역할·게임 상태·규칙 테이블
│   └── NetMessage.swift         호스트↔클라이언트 메시지 (Codable/JSON)
├── Game/
│   └── GameEngine.swift         호스트 전용 권위 게임 엔진 (모든 규칙 판정)
├── Networking/
│   └── MultipeerService.swift   Multipeer 래퍼 (광고/탐색/자동 재접속/다중 세션)
├── ViewModels/
│   └── GameViewModel.swift      호스트·클라이언트 공용 뷰모델
└── Views/                       SwiftUI 화면 (홈/로비/역할/지명/투표/미션/암살/종료)
```

### 설계 포인트

- **스타 토폴로지**: 모든 통신은 호스트를 거칩니다. 규칙 판정은 `GameEngine`에서만 하므로
  클라이언트 간 상태 불일치가 생길 수 없습니다.
- **전체 상태 브로드캐스트**: 변화가 있을 때마다 `PublicGameState` 전체를 보내므로
  메시지가 유실돼도 다음 업데이트로 자연 복구됩니다.
- **비밀 정보 분리**: 역할 등 비공개 정보(`PrivateInfo`)는 각 플레이어에게 **개별 전송**되며
  공개 상태에는 절대 포함되지 않습니다. 게임 종료 시에만 전원 역할이 공개됩니다.

## 커스터마이즈 아이디어

- `GameRules`의 테이블만 바꾸면 인원/원정대 크기 조정 가능
- `Role` enum에 새 역할 추가 (예: 예언자를 아는 호위 역할, 예언자에게 안 보이는 잠복 그림자)
- `Theme.swift`에서 색/무드 변경
